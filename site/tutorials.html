Tutorial 01

Build your first component

1

Create an HTML page

Start with a custom element and load a component module from the pinned CDN.

<dv-counter data-start="10" data-step="5"></dv-counter>
<script type="module" src="https://cdn.jsdelivr.net/gh/kayacuneyd/devinimjs@v0.2.0/dist/modules/dv-counter.js"></script>
2

Change state through UI

The counter reads its attributes once, updates a Proxy-backed state object and morphs only the changed DOM.

Tutorial 02

Pass data and listen for changes

1

Let HTML be the initial API

<dv-counter id="quantity" data-start="1" data-step="1"></dv-counter>
<script type="module">
  document.querySelector('#quantity').addEventListener('dv:change', (event) => {
    console.log(event.detail.count);
  });
</script>
2

Connect this to a backend later

The same event can update a PHP form, LocalStorage or a JSON API. No server technology is required by the component.

Tutorial 03

Render a keyed product list

When list items can move, give each direct sibling a stable data-key. DevinimJS then moves the existing node instead of rewriting the entire list.

template() {
  return html`<ul>${this.state.products.map((product) =>
    html`<li data-key="${product.id}">${product.name}</li>`
  )}</ul>`;
}

Tutorial 04

Share cart state

import { createStore } from 'https://cdn.jsdelivr.net/gh/kayacuneyd/devinimjs@v0.2.0/dist/core.min.js';
export const cart = createStore({ items: [] });

// In a component:
connected() { this.useStore(cart, 'items'); }
add(product) { cart.state.items.push(product); }

Every subscribed component re-renders in the same microtask; path filtering keeps unrelated state changes out.

Tutorial 05

Load data and submit a form

import { createAsyncState, createForm, fetchJson } from 'https://cdn.jsdelivr.net/gh/kayacuneyd/devinimjs@v0.2.0/dist/app.min.js';
const profile = createForm({ email: '' });
await profile.submit((values) => fetchJson('/api/profile.php', {
  method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(values),
}));

Use profile.state.status for loading UI and profile.state.errors for server validation messages.

Tutorial 06

Turn it into a mini application

import { createHashRouter } from 'https://cdn.jsdelivr.net/gh/kayacuneyd/devinimjs@v0.2.0/dist/app.min.js';
const router = createHashRouter().add('/', 'home').add('/products/:id', 'product');
router.subscribe((route) => console.log(route.target, route.params));
router.start();

Link to #/products/42. Hash routes work on shared hosting and static sites because no server rewrite is necessary.
