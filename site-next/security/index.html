Consumer responsibilities

  • Escape backend attributes and treat API responses as untrusted.
  • Sanitize raw HTML and validate URLs before rendering user content.
  • Configure an appropriate Content Security Policy.
  • Pin published assets to a known release.
  • Keep dependencies and server-side integrations updated.

Runtime baseline

DevinimJS escapes interpolated values by default. That behavior does not make unsafe HTML, uploads, authentication or backend endpoints safe automatically; applications remain responsible for their full data flow.
