Application patterns

See the boundary before you copy the pattern.

CKCSS owns presentation. DevinimJS owns browser behavior. The project owns persistence, identity, authorization and trusted business rules.

Backend-dependent production flows

The UI can still be prototyped with fixtures, but production behavior needs an application backend. DevinimJS provides the browser-side request, state and form primitives; it does not provide the server.

Uploads and autosave

The page can call an upload or save handler through fetchJson(); the project must implement validation, storage, access control and persistence.

Common shape

CKCSS       → layout, tokens and visual states
DevinimJS   → components, state, events and async UI
Page/app    → orchestration and persistence adapter
Backend     → identity, data, business rules and trusted writes

For PHP, Laravel, WordPress or another server, print escaped data-* attributes and load the browser module. For a static prototype, replace the backend with fixtures or browser storage and label that boundary clearly.
