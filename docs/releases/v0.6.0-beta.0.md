# v0.6.0-beta.0 release checklist

## Scope

- `component()` is the experimental default authoring path for ordinary components.
- `.dv.js` is ordinary browser ESM; it has no compiler or server dependency.
- The class API remains supported for advanced control.

## Publish order

1. Run `npm run build` and `npm run verify` from a clean checkout.
2. Check every new framework component with `npm run validate:component -- tag-name`.
3. Review `dist/authoring.min.js`, `dist/modules/dv-counter.js` and `dist/modules/dv-search.js`.
4. Commit, tag `v0.6.0-beta.0`, push the tag and wait for CDN propagation.
5. Change public CDN examples from `v0.5.1` only after the tag resolves.
6. Re-run the live-domain checks in `docs/project-memory.md` after deployment.

## Compatibility

- Existing `BaseComponent`, `data-on:event` directives, component tags and `dv:*` events remain
  supported.
- Shared hosting remains supported: copy the built files or serve the source ESM files with the
  JavaScript MIME type.
